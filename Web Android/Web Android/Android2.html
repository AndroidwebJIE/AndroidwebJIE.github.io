¿Qué el SO Android?

Android es un Sistema Operativo basado en Linux, un código abierto especialmente diseñado para dispositivos móviles de pantalla táctil.

Android posee su propia fuente tipográfica, llamada Roboto está diseñada para agilizar y clarificar la lectura en los dispositivos móviles. Otra curiosidad de nuestro SO es que cada versión comienza por un nombre de un postre y va ordenado con la primera letra nombre alfabéticamente.

El SO de Android es líder de mercado frente a iOS de apple, teniendo una cuota de mercado del 71'93% según Statista.

Arquitectura

Kernel de Linux

La base de la plataforma Android es el kernel de Linux. Por ejemplo, el tiempo de ejecución de Android (ART) se basa en el kernel de Linux para funcionalidades subyacentes, como la generación de subprocesos y la administración de memoria de bajo nivel. El uso del kernel de Linux permite que Android aproveche funciones de seguridad claves y, al mismo tiempo, permite a los fabricantes de dispositivos desarrollar controladores de hardware para un kernel conocido.

Capa de abstracción de hardware HAL

Nos ofrece interfaces estándar que muestran las capacidades del hardware del dispositivo a la API de desarrollo de Java. La HAL son varios módulos de biblioteca y cada uno de estos implementa una interfaz para un tipo específico de componente de hardware, como el módulo de la cámara o de Bluetooth. Cuando el marco de trabajo de una API realiza una llamada para acceder a hardware del dispositivo, el sistema Android carga el módulo de biblioteca para el componente de hardware en cuestión.

Tiempo de ejecución

Actualmente Android usa ART (Android Runtime), que es un entorno de ejecucion de apps.

Anteriormente se usaba Dalvik, una máquina virtual, que utilizaba el JIT "just-in-time", un compilador que compila el código cada vez que se inicia la aplicación. En cambio, su predecesor ART convierte el algoritmo llamado AOT "ahead-of-time", un compilador que guarda el código ya compilado en un archivo conforme se descarga la aplicación, así no se tiene que estar compilando el código a cada rato reduciendo el uso del procesador y aumenta la duración de la batería.

Además ART nos trae mejoras de rendimiento, recolección de basura, aplicaciones de depuración y perfilado.

Bibliotecas C/C++ nativas

El ART y HAL usan bibliotecas de código C o C++, la plataforma de Android ofrece una API de marco de trabajo de Android con bibliotecas nativas a las apps. Si requieres escribir en código C/C++ puedes usar el NDK de Android para acceder a dichas bibliotecas.

Marco de trabajo de la API de Android

Todo el conjunto de funciones del SO Android está disponible mediante API escritas en el lenguaje Java. Dichas APIs son las básicas necesarias para poder crear apps de Android simplificando el uso de los componentes del sistema y servicios:

  • Un sistema de vista enriquecido y extensible disponible para compilar la IU de una app
  • Un administrador de recursos que te brinda acceso a recursos sin código, como strings localizadas, gráficos y archivos de diseño.
  • Un administrador de notificaciones que permite que todas las apps muestren alertas personalizadas en la barra de estado.
  • Un administrador de actividad que administra el ciclo de vida de las apps y proporciona una pila de retroceso de navegación común.
  • Proveedores de contenido que permiten que las apps accedan a datos desde otras apps, como la app de Contactos, o compartan sus propios datos.

Apps del sistema

Android incluye un conjunto de apps base de sistema como calendarios, correo electrónico, SMS, navegador, contactos, y herramientas varias.

Accesibilidad

Aquí están características del sistema operativo que permiten la creación y utilización de diferentes ayudas para mejorar la accesibilidad al usuario, han ido apareciendo en diferentes versiones:

Text to Speech
Es el motor que reproduce texto convirtiéndolo a síntesis de voz.
Explore by Touch
Permite al sistema generar eventos de accesibilidad según se vaya colocando el dedo en diferentes partes de la pantalla. Estos eventos son atendidos por un lector de pantalla que se encarga de reproducir el contenido del elemento.
Talk Back
Lector de pantalla que usa Text to Speech.
KickBack
Realiza una vibración al salir de una ventana o al desbloquear pantalla.
SoundBack
Genera un pequeño sonido al realizar una acción sobre el dispositivo.
Tecla Access
Es una entrada alternativa para personas con movilidad reducida, permite conectar diversos dispositivos de entrada.
Dasher
Es un método de entrada, mediante el cúal deslizando suavemente el dedo por la pantalla podremos escribir.
EyesFree Keyboard
Es un método de entrada pensado para personas ciegas ya que mediante un controlador podremos ir leyendo letra a letra para escribir.
BrailleBack
Una app que permite la conexión de un dispositivo externo de Braille.
